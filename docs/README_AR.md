## 1. نظرة عامة على النظام

نظام إدارة المدارس (SchoolMS) هو نظام شامل متعدد المدارس (Multi-Tenant) مبني على:
- **.NET 8** مع **ASP.NET Core MVC**
- **Entity Framework Core** لإدارة قاعدة البيانات
- **ASP.NET Identity** لإدارة المستخدمين والمصادقة
- **SQL Server** كقاعدة بيانات
- **SignalR** للمحادثة والإشعارات الفورية

### أنواع المستخدمين
| النوع | الوصف |
|-------|-------|
| **SuperAdmin** | مدير المنصة - يشرف على جميع المدارس |
| **SchoolAdmin** | مدير المدرسة - يدير مدرسة واحدة |
| **BranchAdmin** | مدير فرع - يدير فرع واحد |
| **Teacher** | معلم |
| **Staff** | موظف |
| **Student** | طالب |
| **DataEntry** | مدخل بيانات |

## 2. بنية النظام التقنية

```
SchoolMS/
├── SchoolMS.Domain          ← الكيانات والواجهات الأساسية
├── SchoolMS.Application     ← خدمات الأعمال و DTOs
├── SchoolMS.Infrastructure  ← قاعدة البيانات والمستودعات
├── SchoolMS.Web             ← واجهة المستخدم (Razor Views + Controllers)
└── SchoolMS.API             ← واجهة البرمجة REST API
```

### طبقات النظام:
- **Domain**: الكيانات (Entities)، الواجهات (Interfaces)، التعدادات (Enums)
- **Application**: الخدمات، DTOs، AutoMapper
- **Infrastructure**: DbContext، Repository، Identity، خدمات البنية التحتية
- **Web**: Controllers، Views، Middleware، Filters

## 3. نظام المنصة - إدارة المدارس المتعددة

### الوصول
- **الرابط**: `/Platform`
- **المتاح لـ**: SuperAdmin فقط

### لوحة تحكم المنصة (`/Platform`)
تعرض:
- إجمالي المدارس والمدارس النشطة
- إجمالي الطلاب والمعلمين عبر جميع المدارس
- الاشتراكات النشطة والاشتراكات التي تنتهي قريباً
- آخر المدارس المضافة

### إدارة المدارس (`/Platform/Schools`)
- **عرض جميع المدارس**: جدول يعرض اسم المدرسة، الرمز (Slug)، عدد الفروع، الطلاب، المعلمين، الخطة الحالية
- **إنشاء مدرسة جديدة**: إنشاء مدرسة + حساب مدير + صلاحيات + فرع افتراضي
- **تعديل المدرسة**: تعديل الاسم والعنوان والحالة
- **تفعيل/تعطيل**: تبديل حالة المدرسة بين نشط/غير نشط
- **إدارة الاشتراك**: تعيين خطة اشتراك لكل مدرسة

### خطط الاشتراك (`/Platform/Plans`)
- إنشاء وتعديل خطط الاشتراك
- كل خطة تحتوي على:
  - **اسم الخطة**
  - **السعر**
  - **المدة** (بالأشهر)
  - **أقصى عدد مستخدمين** (MaxUsers)
  - **أقصى عدد طلاب** (MaxStudents)

## 4. إنشاء مدرسة جديدة

### الخطوات:
1. سجل دخول كـ SuperAdmin
2. اذهب إلى `/Platform/Schools`
3. انقر **"إضافة مدرسة"**
4. أدخل البيانات:

| الحقل | مثال | ملاحظات |
|-------|------|---------|
| اسم المدرسة | مدرسة المستقبل | مطلوب |
| الرمز (Slug) | future-school | مطلوب - يُستخدم في رابط الدخول |
| العنوان | شارع التعليم 123 | اختياري |
| تاريخ الانتهاء | 2026-12-31 | اختياري |
| نشط | ✓ | تفعيل/تعطيل المدرسة |
| المنصة الإلكترونية | ✓ | تفعيل الدورات الإلكترونية |

5. أدخل بيانات مدير المدرسة:

| الحقل | مثال | ملاحظات |
|-------|------|---------|
| اسم المدير | أحمد محمد | مطلوب |
| البريد الإلكتروني | admin@future-school.com | يجب أن يكون بريد صحيح |
| كلمة المرور | Admin@1234 | 8+ حروف، حرف كبير، صغير، رقم، رمز خاص |
| خطة الاشتراك | Premium | اختياري |

### ما يحدث تلقائياً عند إنشاء المدرسة:
1. ✅ يتم إنشاء سجل المدرسة
2. ✅ يتم إنشاء فرع افتراضي "Main Branch"
3. ✅ يتم إنشاء حساب مدير المدرسة (دور: SchoolAdmin)
4. ✅ يتم إنشاء جميع الصلاحيات (164 صلاحية)
5. ✅ يتم منح المدير جميع الصلاحيات
6. ✅ يتم ربط المدير بالفرع الافتراضي
7. ✅ يتم تعيين خطة الاشتراك (إذا تم اختيارها)

## 5. نظام الاشتراكات والحدود

### التحقق التلقائي عند كل طلب (Middleware):
عند كل طلب HTTP من مستخدم مسجل:

```
المستخدم ← التحقق من أن المدرسة نشطة
       ← التحقق من عدم انتهاء الاشتراك
       ← إذا انتهى → إعادة التوجيه إلى صفحة "انتهى الاشتراك"
```

### التحقق عند تسجيل الدخول:
- ✅ التحقق من وجود المدرسة
- ✅ التحقق من أن المدرسة نشطة
- ✅ التحقق من عدم انتهاء الاشتراك
- ✅ رسالة خطأ واضحة لكل حالة

### التحقق من الحدود:
- ✅ عند **إضافة طالب**: يتم التحقق من `MaxStudents` في خطة الاشتراك
- ✅ عند **إضافة مستخدم**: يتم التحقق من `MaxUsers` في خطة الاشتراك
- ✅ إذا تم تجاوز الحد → رسالة خطأ واضحة

### حالات خاصة:
- **بدون خطة اشتراك**: لا توجد حدود (غير محدود)
- **SuperAdmin**: يتجاوز جميع عمليات التحقق
- **صفحة انتهاء الاشتراك**: `/subscription-expired`

## 6. روابط تسجيل الدخول لكل مدرسة

كل مدرسة لها رابط دخول فريد بناءً على الـ Slug:

### تنسيق الرابط:
```
https://your-domain.com/school/{slug}
```

### أمثلة:
| المدرسة | الرمز (Slug) | رابط الدخول |
|---------|-------------|-------------|
| مدرسة المستقبل | future-school | `/school/future-school` |
| مدرسة النجاح | success-school | `/school/success-school` |
| Demo School | demo-school | `/school/demo-school` |

### تسجيل دخول SuperAdmin:
- الرابط: `/Account/Login`
- اترك حقل "رمز المدرسة" فارغاً
- أدخل البريد الإلكتروني وكلمة المرور

### تسجيل دخول مدير/موظف مدرسة:
- الرابط: `/school/{slug}` أو `/Account/Login`
- أدخل رمز المدرسة (Slug) في حقل "رمز المدرسة"
- أدخل البريد الإلكتروني وكلمة المرور

## 7. عزل البيانات بين المدارس

### كيف يعمل العزل:
النظام يستخدم **Global Query Filters** في Entity Framework:

```csharp
// كل كيان يحتوي على SchoolId
builder.Entity<T>().HasQueryFilter(e => !e.IsDeleted && e.SchoolId == currentSchoolId);
```

### هذا يعني:
- ✅ عند تسجيل الدخول بمدرسة رقم 1 → تظهر فقط بيانات المدرسة رقم 1
- ✅ عند تسجيل الدخول بمدرسة رقم 2 → تظهر فقط بيانات المدرسة رقم 2
- ✅ لا يمكن لأي مدرسة رؤية بيانات مدرسة أخرى
- ✅ الطلاب، المعلمون، الفروع، الصفوف، المواد — كل شيء معزول

### عند إنشاء سجل جديد:
```csharp
// يتم تعيين SchoolId تلقائياً من المستخدم الحالي
if (entry.Entity.SchoolId == 0)
    entry.Entity.SchoolId = currentSchoolId;
```

### SuperAdmin:
- يرى بيانات المدرسة المرتبط بها (SchoolId = 1)
- يمكنه الوصول لـ `/Platform` لإدارة جميع المدارس
- صفحات المنصة تستخدم `IgnoreQueryFilters()` لعرض جميع البيانات

## 8. نظام الصلاحيات

### البنية:
- **Permission**: صلاحية = (صفحة + إجراء)
  - مثال: `Students + View` = عرض الطلاب
  - مثال: `Students + Add` = إضافة طالب
  - مثال: `Students + Edit` = تعديل طالب
  - مثال: `Students + Delete` = حذف طالب
- **UserPermission**: ربط المستخدم بالصلاحيات

### الصفحات المتاحة (41 صفحة × 4 إجراءات = 164 صلاحية):
```
Dashboard, Students, Teachers, Staff, Subjects, Divisions, Grades,
ClassRooms, TeacherAssignments, Salaries, Expenses, Installments,
ExamSchedule, WeeklySchedule, StudentGrades, Branches, Leaves,
Attendance, Notifications, Users, Promotion, Homework, Quizzes,
Carousel, OnlinePlans, OnlineSubscriptions, PromoCodes, Courses,
LiveStreams, Chat, Parents, Events, Announcements, Behavior,
Health, Complaints, Visitors, Library, Transport, Assets, AuditLog
```

### الإجراءات لكل صفحة:
| الإجراء | الوصف |
|---------|-------|
| View | عرض الصفحة والبيانات |
| Add | إضافة سجل جديد |
| Edit | تعديل سجل موجود |
| Delete | حذف سجل |

### التحقق من الصلاحيات:
- يتم عبر `[HasPermission("PageName", "Action")]` على كل Controller Action

## 11. التحقق من الاشتراك والحدود

### Middleware التحقق (`SubscriptionValidationMiddleware`):
يعمل على **كل طلب HTTP** بعد المصادقة:

```
1. هل المستخدم SuperAdmin؟ → تجاوز (لا تحقق)
2. هل المدرسة نشطة؟ → إذا لا → صفحة "المدرسة معطلة"
3. هل الاشتراك نشط؟ → إذا انتهى → صفحة "انتهى الاشتراك"
4. تخزين حدود الخطة في الطلب
```

### تحقق الحدود (`SubscriptionLimitService`):
| التحقق | متى يعمل | الرسالة |
|--------|----------|---------|
| MaxStudents | عند إضافة طالب جديد | "تم الوصول للحد الأقصى لعدد الطلاب" |
| MaxUsers | عند إضافة مستخدم جديد | "تم الوصول للحد الأقصى لعدد المستخدمين" |
| Expiry | عند تسجيل الدخول وكل طلب | "انتهى اشتراك مدرستك" |

### مثال عملي:
```
خطة "Premium": MaxStudents=500, MaxUsers=50, Duration=12 شهر
→ المدرسة لديها 499 طالب → يمكن إضافة طالب واحد
→ المدرسة لديها 500 طالب → لا يمكن إضافة طالب ← رسالة خطأ
→ انتهت 12 شهر → جميع المستخدمين يُعاد توجيههم لصفحة "انتهى الاشتراك"
```

## 12. دعم اللغات

### اللغات المدعومة:
- **العربية** (ar) - اتجاه من اليمين لليسار (RTL)
- **الإنجليزية** (en) - اتجاه من اليسار لليمين (LTR)

### تبديل اللغة:
- يوجد زر تبديل في شريط العلوي
- يتم حفظ الاختيار في Cookie لمدة سنة
- يعمل عبر `/Language/SetLanguage?culture=ar`

### ملفات الترجمة:
```
Resources/SharedResource.en.resx  ← الإنجليزية
Resources/SharedResource.ar.resx  ← العربية
```

### الاستخدام في الواجهات:
```razor
@Localizer["Dashboard"]    ← تعرض "Dashboard" أو "لوحة التحكم"
@Localizer["Students"]     ← تعرض "Students" أو "الطلاب"
```

## 15. بنية قاعدة البيانات

```
Schools                    ← المدارس

AcademicYears              ← السنوات الدراسية

```

```
CreatedAt       ← تاريخ الإنشاء
CreatedBy       ← من أنشأه
UpdatedAt       ← تاريخ التعديل
UpdatedBy       ← من عدّله
DeletedAt       ← تاريخ الحذف
DeletedBy       ← من حذفه
```

## 16. API واجهة البرمجة

### المصادقة:
```
POST /api/Auth/login
Body: { "email": "...", "password": "...", "schoolSlug": "..." }
Response: { "token": "JWT_TOKEN" }
```

### استخدام التوكن:
```
Authorization: Bearer JWT_TOKEN
```

### أمثلة نقاط النهاية:
```
GET    /api/Students          ← جلب الطلاب
POST   /api/Students          ← إنشاء طالب
PUT    /api/Students           ← تعديل طالب
DELETE /api/Students/{id}      ← حذف طالب

GET    /api/Teachers           ← جلب المعلمين
GET    /api/Branches           ← جلب الفروع
GET    /api/Dashboard          ← بيانات لوحة التحكم
```

---

## ⚡ ملخص سريع للبدء

### 1. تشغيل النظام:
```bash
cd src/SchoolMS.Web
dotnet run
```

### 2. أول دخول كـ SuperAdmin:
```
الرابط: https://localhost:xxxx/Account/Login
البريد: admin@demo-school.com
كلمة المرور: Admin@1234
```

### 3. إنشاء مدرسة جديدة:
```
اذهب إلى: /Platform/Schools → إضافة مدرسة
```

### 4. تسجيل دخول المدرسة الجديدة:
```
الرابط: https://localhost:xxxx/school/{slug}
```

---

> **ملاحظة**: النظام يعمل بنظام الحذف الناعم (Soft Delete) — لا يتم حذف أي بيانات نهائياً بل يتم تعليمها كمحذوفة.

> **ملاحظة**: جميع الأوقات تُخزن بتوقيت UTC.
